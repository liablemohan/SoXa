# SoXa Development Roadmap (5 Stages)

## Stage 1: Repository MVP (8–12 Weeks)

### Objective

Transform the current static website into a functional Sanskrit research repository.

### Features

#### Backend

* Django + Django REST Framework setup
* PostgreSQL database integration
* Dockerized development environment

#### Repository Modules

* Thesis Repository
* Epigraphical Repository
* Literary History Repository

#### Database

Implement schemas for:

* Users
* Theses
* Epigraphical records
* Literary records

#### Search

* Basic search
* Filter by year
* Filter by institution
* Filter by subject
* Pagination
* Sorting

#### File Handling

* Upload PDFs
* Download files
* Access-level badges

#### Admin

Use Django Admin for:

* Add/Edit/Delete records
* Bulk imports using CSV

### Deliverables

* Live repository website
* Admin dashboard
* Searchable records
* Import pipeline for legacy datasets

# Priority Principle

The order should always be:

**Content → Discovery → Community → Contribution → Infrastructure**

Do not begin with advanced AI, recommendation systems, or Elasticsearch clusters.

First ensure that:

1. Records exist,
2. Scholars can find them,
3. Scholars join the platform,
4. Scholars contribute,
5. The ecosystem scales.

This sequence minimizes cost, maximizes early impact, and aligns with SoXa's vision of becoming a community-driven Sanskrit knowledge platform.
